プロジェクト構造をMarkdownで記載し、各ディレクトリの説明もコメントとして追加します。

```markdown
# プロジェクト構造

<!-- プロジェクトのルートディレクトリ -->
Project_root/
├── README.md                     <!-- プロジェクトの概要説明ファイル -->
├── CB02外環検査/                 <!-- CB02外環検査関連のメインディレクトリ -->
│   ├── data/                     <!-- データ格納ディレクトリ -->
│   │   ├── input/               <!-- 入力データ用ディレクトリ -->
│   │   └── output/              <!-- 出力データ用ディレクトリ -->
│   ├── notebook/                 <!-- Jupyter notebook格納ディレクトリ -->
│   │   ├── Past/                <!-- 過去の分析用ノートブック -->
│   │   └── Present/             <!-- 現在使用中のノートブック -->
│   ├── report/                   <!-- レポート出力用ディレクトリ -->
│   └── src/                      <!-- ソースコード格納ディレクトリ -->
│
└── 渦流探傷外環検査/             <!-- 渦流探傷外環検査関連のメインディレクトリ -->
    ├── data/                     <!-- データ格納ディレクトリ -->
    │   ├── input/               <!-- 入力データ用ディレクトリ -->
    │   └── output/              <!-- 出力データ用ディレクトリ -->
    └── notebook/                 <!-- Jupyter notebook格納ディレクトリ -->

```

このディレクトリ構造について、以下のような補足説明を追加することもできます：

```markdown
## ディレクトリ構成の説明

### CB02外環検査
- `data/input`: 元データや外部から入手したデータを格納
- `data/output`: 処理結果や分析結果のデータを格納
- `notebook/Past`: これまでの分析で使用した過去のノートブックを保管
- `notebook/Present`: 現在使用中・開発中のノートブックを格納
- `report`: 分析結果のレポートや図表を格納
- `src`: 共通で使用する関数やクラスなどのソースコードを格納

### 渦流探傷外環検査
- `data/input`: 渦流探傷検査の元データを格納
- `data/output`: 渦流探傷検査の分析結果を格納
- `notebook`: 渦流探傷検査の分析用ノートブックを格納
```
